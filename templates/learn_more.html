How to Use the Timetable Generator

This page explains how to correctly use each feature of the Timetable Generation System. Please read this before entering data to avoid incorrect or incomplete timetable generation.

1. General Timetable Inputs

These inputs define the basic structure of your academic day.

  • Start Time: Time at which the first lecture begins (HH:MM format)
  • End Time: Time at which the academic day ends
  • Working Days: Number of working days in a week (usually 5 or 6)
  • Periods per Day: Total number of lecture slots per day
Make sure the total duration of lectures and breaks fits between start and end time.

2. Lecture, Practical, and Break Durations

These values define how time is distributed throughout the day.

  • Lecture Duration: Duration of one theory lecture (in minutes)
  • Practical Duration: Duration of one practical session
  • Practical Batches Number of Practical Batches per Division
  • Lunch Break: Duration of lunch break
  • Short Break: Duration of short breaks between lectures
  • Short Break Count: Number of short breaks per day

3. Faculty Data Extraction (College Website Link)

To automatically extract faculty details, the system requires a link to your college’s official faculty information page.

What link should be provided?

  • The link must be from the official college website
  • The page should list faculty names and departments
  • Preferred pages include:
    • Department faculty listing page
    • Staff directory page
Example: https://college.edu/computer-engineering/faculty
Do not provide login-protected pages, Google Drive links, or scanned images.

4. Syllabus PDF Upload

The syllabus PDF is used to extract subject names and academic structure.

Accepted PDF Type

  • Official university or college syllabus PDF
  • Text-based PDF (not scanned images)
  • Should clearly mention subject names, semester, and theory/practical details
Recommended file name: CE_Sem4_Syllabus.pdf

5. Subject and Faculty Assignment

  • Assign one or more faculty members per subject
  • Specify theory, practical, and tutorial hours
  • The system automatically checks faculty collisions

6. Generating the Timetable

  • Click Create Timetable after all inputs are filled
  • Division-wise timetables are generated
  • Faculty conflicts are avoided automatically

7. Chatbot Usage

  • “Who teaches Operating Systems on Tuesday?”
  • “Show timetable for Division 1”
  • “When is the next practical session?”
Chatbot responses are generated using the current timetable data.
